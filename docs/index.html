What Stage327 Adds

vulnerability_type

Standard classification such as prompt-injection, sensitive-information-disclosure, or supply-chain.
target

The verified target: model-output, repository, commit, file, line-range, artifact, or workflow.
ai_claim_id

Identifier for the AI-generated vulnerability claim.
reproduction_id

Identifier for the reproduction evidence.
result

reproduced, not-reproduced, partial, or inconclusive.
evidence_files

Files used as evidence, such as prompt.txt, response.txt, and run.log.
sha256

Hash dictionary that binds every evidence file to an integrity value.

Core Question

Can the AI vulnerability reproduction result be represented as structured,
hash-bound, third-party-verifiable audit evidence?

Stage326 vs Stage327

Stage326

Checks whether the AI claim and reproduction result refer to the same target.
Stage327

Defines the standard format of the reproduction evidence itself.
Stage326 = Same target verification
Stage327 = Standardized reproduction evidence
